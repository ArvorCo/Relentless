Why Relentless?

Because your AI agent shouldn't quit after one try. Relentless keeps going until the job is done.

The Ralph Wiggum Pattern, Evolved

Inspired by Geoffrey Huntley's brilliant idea: keep spawning fresh AI agents until the work is done. We took it further with multi-agent support, smart routing, and cost optimization.

What's in a Name?

According to Matt Pocock, if you're trying to sneak Ralph into the enterprise, just pretend it's an acronym:

Relentless Agentic Looping Programming Helper

So naturally, RELENTLESS is the recursive acronym:

RALPH Endlessly Navigating Tasks, Learning, Evolving, Solving, Scaling

It's recursive. We loop. That's kind of our thing.

How It Works

Each iteration spawns a fresh agent with clean context. Memory persists through Git and tracking files.

1

Fresh Context

New agent instance spawned with clean context window. No accumulated confusion.

2

Load Constitution

Agent reads project coding principles and governance rules. Stays on track.

3

Find Next Story

Agent reads PRD, finds next incomplete story with all dependencies met.

4

Implement & Commit

Agent implements the story, runs tests, commits changes to Git.

5

Update Progress

Sets passes: true for completed story. Adds learnings to progress.txt.

6

Loop Until Done

Process repeats until all stories pass. Memory persists via Git history.

Get Started

Install Relentless and start automating your AI coding workflow in minutes.

# Install globally npm install -g @arvorco/relentless # Initialize in your project cd your-project relentless init # Create spec and run (Claude Code / Amp) /relentless.specify Add user authentication with OAuth2 /relentless.plan /relentless.tasks # Convert and run with beautiful TUI relentless convert relentless/features/001-auth/tasks.md --feature 001-auth relentless run --feature 001-auth --tui
